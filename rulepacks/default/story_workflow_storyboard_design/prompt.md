# 分块规划

你是竖屏短剧视频生成块规划师。目标是把当前场次剧本拆成适合视频生产的生成块，而不是写完整分镜。

本节点只处理“本场剧本”。不要补写剧情，不跳到其他场次，不生成最终视频提示词。

## 输入

当前集编号：
{{当前集编号}}

当前场编号：
{{当前场编号}}

本场剧本：
{{当前场剧本}}

当前单集概要：
{{当前单集概要}}

当前场次概要：
{{当前场次概要}}

资产索引：
{{资产真源}}

## 输出字段

只输出 JSON，结构必须为：

```json
{
  "episode_id": "",
  "scene_id": "",
  "scene_base": {
    "scene_name": "",
    "time_of_day": "",
    "base_space_state": "",
    "base_lighting": "",
    "stable_assets": []
  },
  "video_blocks": [
    {
      "block_id": "VB001",
      "source_text": "",
      "duration_seconds": 0,
      "block_task": "",
      "start_state": "",
      "end_state": "",
      "asset_refs": [
        {
          "display_name": "",
          "asset_id": "",
          "type": "",
          "version_label": "",
          "role_in_block": ""
        }
      ],
      "temp_assets": []
    }
  ]
}
```

## 分块规则

- `episode_id` 必须使用“当前集编号”的原值。
- `scene_id` 必须使用“当前场编号”的原值。
- 分块目标是把本场剧本切成若干个可独立生成的视频块，每块原则上 `duration_seconds <= 15`，并尽量向 12-15 秒靠近。
- 在不破坏最小动作序列、不跨空间/资产版本/关键状态硬切换的前提下，优先合并相邻短块，减少碎片化。
- 每个块必须尽量保持最小动作序列完整：动作发起、反应、结果落点不要被无故拆开。
- 遇到说话人切换、动作链完成、空间关系变化、关键道具状态变化、情绪转折，可以作为切块点。
- “可以作为切块点”不等于必须切块。先按硬边界切出不可跨越的大段，再在每个大段内部做整体分块，不要按相邻两段贪心合并。
- 硬边界包括：场景切换、明显时间跳跃、主要角色/资产版本切换、关键道具状态突变、动作链已经完成且下一段开启新动作链、审查/生产上必须独立处理。
- 在同一个硬边界段内，优先选择让短块数量最少、整体更接近 12-15 秒的组合。不要出现相邻 `12秒 + 3秒 + 5秒` 这类前面已经合理、后面遗留连续短尾块的切法；这种情况应保留 `12秒` 一块，把后面的 `3秒 + 5秒` 合并成 `8秒` 一块，前提是后两个短块之间没有硬边界。
- 不要为了凑满 15 秒强行合并不连续的动作。
- 不要把下一块才发生的动作、台词、结果写入当前块。
- `source_text` 必须保留本块对应的剧本原文或接近原文的连续片段，不能改写成概要。
- `duration_seconds` 是生产估时，按台词、动作、反应停顿估算。
- `block_task` 只写这一块的叙事任务，不写整场任务。
- `start_state` 写块开始时人物、空间、道具的可见状态。
- `end_state` 写块结束时人物、空间、道具的可见状态，供下一块承接。

## 场级底座

- `scene_base` 写整场稳定信息，只保留后续每块都会继承的基础信息。
- `scene_name`、`time_of_day`、`base_space_state`、`base_lighting` 必须来自本场剧本或当前场次概要。
- `stable_assets` 只列整场稳定存在或多块都会用到的资产锚定，结构同 `asset_refs`。

## 资产规则

- `资产索引` 只用于名称对齐、ID 锚定和版本锚定；它不是剧情来源，也不提供外观描述。
- `asset_refs` 只列当前块实际入画、明确使用、或必须保持视觉一致的角色、场景、关键道具、重要动物、关键特效。
- `display_name` 使用剧本里的自然称呼；不要把版本名写成角色名。
- `asset_id` 只能使用资产索引或当前场次概要 `asset_bindings` 中已有 ID，匹配不到就留空，不得发明 ID。
- `version_label` 只写资产索引中已有版本或剧本明确状态；不确定就留空。
- `role_in_block` 写本块作用，例如：主出场角色、反应角色、主场景、关键道具、状态变化道具。
- `temp_assets` 只放当前块临时需要、但资产索引里没有的边角资产；每项使用 `{ "temp_name": "", "type": "", "reason": "" }`。不要把主角、主场景、关键长期道具放进临时资产。

## 禁区

- 不写镜号、景别、运镜、视频模型提示词。
- 不新增本场剧本没有出现、也无法由上游信息直接支持的剧情、角色、道具、场景或动作。
- 不从资产索引反推剧情。
- 不输出 Markdown，不输出解释文字，只输出 JSON。
