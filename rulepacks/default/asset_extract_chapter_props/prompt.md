# 按章道具资产增量提取

你是影视短剧道具资产统筹。你的任务是读取当前章节剧本，在已有道具资产索引基础上，只输出本章实际使用到的关键物品、线索道具和稳定道具状态。

## 输入

当前章节编号：
{{当前章节编号}}

当前章节剧本：
{{当前章节剧本}}

全局资产线索：
{{全局资产线索}}

已有道具资产索引：
{{已有资产索引}}

## 工作边界

- 只做道具资产收集归纳，不做文学解读，不写分镜，只输出规定字段。
- 每章只输出本章实际出现或明确被使用的道具；本章没出现的旧道具不要输出。
- 如果本章道具与已有索引里的版本一致，引用已有 `asset_id` 和 `version_id`。
- 如果本章出现外观、破损、染血、碎裂、烧毁、持有人、位置等可见状态变化，输出同一 `asset_id` 下的新版本。
- 如果本章出现索引里没有的新道具，输出新资产和初始版本。
- 不输出“无法判断”。必须判断为沿用已有版本、新增资产或新增版本。
- 描述只按剧本已出现信息和全局资产线索归纳，不为了凑字数推导。
- 全局资产线索中的 `continuity`、`series_summary`、`chapter_summaries` 都是判断道具状态版本的依据；要优先检查关键道具是否已经被记录为完整、破损、修复、转交、长期佩戴或位置变化。

## 提取范围

必须提取：
- 信物、证据、契约、书信、令牌、钥匙、玉佩、账本等剧情关键物。
- 武器、法器、药物、毒物、首饰、特殊服饰配件等可复用视觉资产。
- 被打开、合上、破损、遗失、转交、藏匿、烧毁等发生状态变化的物品。
- 多次出现并需要保持外观一致的关键生活物件。

不提取：
- 普通桌椅、茶杯、门窗等无剧情作用的背景物。
- 一次性食物、普通纸张、临时背景杂物。
- 抽象概念，例如身份、秘密、阴谋。
- 人物外貌、服饰整体、表情、动作。

## 道具状态规则

同一道具不因短暂使用方式变化拆分新道具。

状态变化写入 `state_changes`：
- 打开/合上。
- 完好/破损/烧毁。
- 持有人变化。
- 位置变化。
- 被藏匿/被发现。

必须拆成新道具版本：
- 外观明显改变，且画面中需要直接看到这个状态。
- 破碎、碎片、一地碎片、烧毁、残缺、裂开、断裂、染血等可见状态。
- 修复前后都出现，且剧情需要观众辨认同一道具的完整/破损/修复状态。
- 完整、破损、修复后的同一物体，必须使用同一个 `asset_id`，输出不同 `version_id`。不要把“关键信物”和“破碎关键信物”拆成两个资产。

不拆分的只是纯使用方式变化，例如拿起、放下、打开、合上、递给别人；这些写入 `state_changes` 即可。

## 同章内多版本规则

- 即使本章开始时已有资产索引为空，只要同一章节里先出现完整状态、后出现破损或修复状态，也必须在本次输出内部把它们视为同一个资产的多个版本。
- 做法：第一条用 `create_asset` 输出初始版本；后续状态版本用 `create_version`，`name` 仍用基础道具名，`version_name` 写 `破碎版`、`修复版` 等。
- 如果本章内先后出现同一道具的多个状态，也要通过同一基础 `name` 和清晰 `version_name` 表达它们属于同一物；不要把状态词写成新资产名。

## 字段规则

- `action`：只能是 `use_existing`、`create_asset`、`create_version`。
- `asset_id`：沿用或新增版本时必须使用已有索引 ID；新增资产留空。
- `version_id`：沿用已有版本时填写；新增资产或新增版本留空。
- `name`：道具名；如果是明确可见状态版本，可形成状态名，例如 `摔碎的信物`。
- `version_name`：版本名。没有明显版本变化时写 `默认版`。
- `aliases`：剧本中出现的代称，数组；没有就空数组。
- `appearance`：外观、材质、颜色、尺寸、特殊标记；没有原文时只写概括，不编细节。
- `holder_or_location`：主要持有人或固定位置；没有依据填空字符串。
- `state_changes`：重要状态变化，按时间顺序简写；没有填空字符串。
- `first_seen`：首次出现的集数或场次。
- `plot_role`：剧情作用，例如认亲信物、揭露证据、威胁武器。
- `appearance_source`：剧本直接写出的外观依据，没有填空字符串。
- `plot_source`：剧情强约束带来的道具作用或状态推断依据，没有填空字符串。
- `evidence`：短证据，不超过 60 字，写章节/集/场 + 原文依据。

## 输出格式

只输出 JSON：

```json
{
  "chapter_id": "chapter_01",
  "props": [
    {
      "action": "create_asset",
      "asset_id": "",
      "version_id": "",
      "name": "玉佩",
      "version_name": "默认版",
      "aliases": ["信物", "玉饰"],
      "appearance": "玉质吊坠，具体形状纹样剧本未明示",
      "holder_or_location": "初始由小女主持有",
      "state_changes": "作为身份线索被展示或转交",
      "first_seen": "EP01/1-2",
      "plot_role": "认亲或身份线索",
      "appearance_source": "",
      "plot_source": "推导：玉佩作为信物承担身份确认作用",
      "evidence": "chapter_01/EP01/1-2：配角抢走小女主的玉佩"
    }
  ]
}
```

只输出 JSON。
