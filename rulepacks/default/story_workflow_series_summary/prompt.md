# 全集概要

当前节点由后端程序机械合并 剧情地图/角色概要/信息连续性 生成，不默认调用 LLM。

## 程序合并原则

- 不理解剧情。
- 不判断冲突。
- 不重写内容。
- 不排序筛选。
- 只把 剧情地图/角色概要/信息连续性 的固定字段搬运到 全集概要。

## 输出字段

- `series_bible_summary`：来自 剧情地图。
- `chapter_map`：来自 剧情地图。
- `character_summary`：来自角色概要。
- `must_track_items`：来自 信息连续性。

## 规则

- 此文件只作为流程配置中的节点说明和备份参考。
- 如果未来重新启用 LLM 全集概要，必须先重新讨论字段边界。

只输出 JSON。
