# 产品与交互设计方案

## 当前阶段目标

当前阶段不是生产系统交付，而是：

- 需求方案确认
- 信息架构确认
- 可扩展前端原型确认
- 核心真源结构确认

## 页面结构

### 基础壳子

- 顶部窄栏：项目、状态、保存、生成、导出、全局工具入口。
- 左侧导航：一级模块，可收起，不承载大段业务信息。
- 右侧工具抽屉：默认收起，打开后覆盖在页面上层，不挤压主工作区。
- 主工作区：只展示当前任务需要人审阅和操作的内容。

### 0. 项目管理

目的：支持多个项目、保存、版本和任务恢复。

当前状态：

- 已有 localStorage 本地项目仓库。
- 支持本地多项目。
- 支持保存版本摘要。
- 顶部显示当前项目和版本数。
- 右侧工具抽屉提供项目列表、新建项目、保存版本。

后续必须补：

- 新建/复制项目
- 最近打开
- 项目级导出
- 完整版本 diff
- 回滚
- 后端持久化

### 1. 剧本校验

目的：确认后续 LLM 使用的剧本源头是可信的。

主区：

- 原始剧本输入
- 题材/导演风格/基础参数
- 导入剧本
- 生成草稿
- 清洗稿
- 审校疑点
- 统计信息
- 写回剧本

### 2. 剧本统筹

目的：在资产审阅前完成全剧统筹和章节概要，让后续资产抽取和分镜统筹有可靠的叙事坐标。

主区：

- 全剧统筹：剧情地图、角色概要、信息连续性、全集概要。
- 章节概要：章节概要、每集标题、每集一句话梗概。
- 运行/重跑/保存。
- 人工审阅和修改。

人类操作重点：确认全剧主线、章节划分、角色弧线、伏笔、资产变化风险、章节功能、每集标题和每集一句话梗概。

这些产物会同时给资产审阅和分镜统筹使用。剧本统筹页不做资产定稿，不写分镜，不生成视频提示词。

### 3. 资产审阅

目的：作为视觉一致性中心。

右侧：

- 生图队列
- 资产描述和提示词编辑
- 首次来源
- 连续性约束
- 候选图
- 锁定版本
- 失败/待处理状态

人类操作重点：补全资产描述、生成候选图、比较候选、锁定最终图。只展示资产但不能操作是不够的。

### 4. 分镜统筹

目的：执行单集概要、场次概要、分块规划，把剧本统筹信息转成可生产的视频块。当前分块规划和视频提示词使用资产真源索引，后端在最终生成阶段按资产 ID 和版本标签查找参考图。

主区：

- 单集概要：单集概要。
- 场次概要：可独立执行，也可合并到 05。
- 分块规划：默认按场执行。
- 场级重跑、单场锁定、分镜审阅。

人类操作重点：确认单集任务、场内连续性、人物站位、道具状态、资产引用和分块边界是否正确。

分镜统筹页必须依赖资产真源 ID 和版本，不允许使用 LLM 临时命名。

### 5. 视频生成

目的：执行视频提示词和后续视频生成检查。

主区：

- 视频提示词。
- 视频块资产包。
- 资产真源图引用。
- 视频候选。
- 人工确认和单块重跑。

人类操作重点：检查视频提示词是否忠于视频块原文，资产包是否正确，视频候选是否可用，是否需要重跑。

视频生成页不直接修改资产真源。如发现资产问题，标记并返回资产审阅修正。

### 6. 流程配置

目的：确认长文本处理、体裁规则、输出适配、重跑粒度。

人类操作重点：

- 项目开始前确认流程模板。
- 换体裁、平台、输出文件规则时改配置。
- 某阶段输出异常时定位是规则包、执行边界还是输入输出问题。

日常分镜/生图生产不应频繁进入这个页面。

## 布局原则

- 左侧只做一级导航，可收起。
- 顶部只放项目、保存、生成、导出、全局工具入口。
- 右侧工具抽屉默认收起，打开后覆盖页面，不挤压主区。
- 剧本校验页在主区显示全文剧本输入。
- 其他页面不重复显示全文剧本。
- 页面必须适应多集、多资产、多版本。
- 页面必须避免卡片嵌套和无意义说明文案。
- 底部执行反馈默认只显示一条状态；展开后作为大排查面板使用。

## 一级页面确认

当前确认的一级页面：

1. 项目管理
2. 剧本校验
3. 剧本统筹
4. 资产审阅
5. 分镜统筹
6. 视频生成
7. 任务记录
8. 流程配置

取消独立“辅助信息”页面。原辅助信息职责已经被分镜工作流吸收：

- 集级辅助进入单集概要。
- 场级连续性进入场次概要。
- 资产锁定项进入资产审阅。
- 镜头级连续性进入分块规划和视频提示词。

## 参考方向

参考大厂 AI 视频工具的工作台式布局，但不照抄：

- 左栏：项目、任务、流程状态
- 中间：当前任务主编辑区
- 右侧或卡片区：上下文、版本、问题、结果

当前原型采用工作台壳子：左侧导航 + 主区 + 右侧覆盖工具抽屉。
