# 产品与交互设计方案

## 当前阶段目标

当前阶段不是生产系统交付，而是：

- 需求方案确认
- 信息架构确认
- 可扩展前端原型确认
- 核心真源结构确认

## 页面结构

### 基础壳子

- 顶部窄栏：项目、状态、保存、生成、导出、全局工具入口。
- 左侧导航：一级模块，可收起，不承载大段业务信息。
- 右侧工具抽屉：默认收起，打开后覆盖在页面上层，不挤压主工作区。
- 主工作区：只展示当前任务需要人审阅和操作的内容。

### 0. 项目管理

目的：支持多个项目、保存、版本和任务恢复。

当前状态：

- 已有 localStorage 本地项目仓库。
- 支持本地多项目。
- 支持保存版本摘要。
- 顶部显示当前项目和版本数。
- 右侧工具抽屉提供项目列表、新建项目、保存版本。

后续必须补：

- 新建/复制项目
- 最近打开
- 项目级导出
- 完整版本 diff
- 回滚
- 后端持久化

### 1. 剧本校验

目的：确认后续 LLM 使用的剧本源头是可信的。

主区：

- 原始剧本输入
- 题材/导演风格/基础参数
- 导入剧本
- 生成草稿
- 清洗稿
- 审校疑点
- 统计信息
- 写回剧本

### 2. 资产生图

目的：作为视觉一致性中心。

右侧：

- 生图队列
- 资产描述和提示词编辑
- 首次来源
- 连续性约束
- 候选图
- 锁定版本
- 失败/待处理状态

人类操作重点：补全资产描述、生成候选图、比较候选、锁定最终图。只展示资产但不能操作是不够的。

### 3. 上下文包

目的：审阅后续阶段必须使用的辅助信息。

右侧展示：

- 03 集级辅助
- 04 场级连续性
- 角色/场景/道具锁定项
- 镜头连续性监看

原则：辅助信息不是信息仓库。必须说明来源、用途和可靠性；不能重复 06/07 当前执行边界已经能看到的信息。

这个页面在协作中的作用是“确认中间约束是否可靠”，不是主要生产入口。

### 3.5 流程配置

目的：确认长文本处理、体裁规则、输出适配、重跑粒度。

人类操作重点：

- 项目开始前确认流程模板。
- 换体裁、平台、输出文件规则时改配置。
- 某阶段输出异常时定位是规则包、执行边界还是输入输出问题。

日常分镜/生图生产不应频繁进入这个页面。

### 4. 分镜工作台

目的：审阅和编辑分镜真源。

右侧展示：

- 本集必须遵守
- 场级规划
- 重跑控制
- 镜头审阅
- 视频提示词预览
- JSON 真源编辑
- 导出本集

人类操作重点：确认场级连续性，审镜头是否符合视听表达，选择场级/块级/提示词级重跑，不在主区重复选集和项目泛统计。

## 布局原则

- 左侧只做一级导航，可收起。
- 顶部只放项目、保存、生成、导出、全局工具入口。
- 右侧工具抽屉默认收起，打开后覆盖页面，不挤压主区。
- 剧本校验页在主区显示全文剧本输入。
- 其他页面不重复显示全文剧本。
- 页面必须适应多集、多资产、多版本。
- 页面必须避免卡片嵌套和无意义说明文案。
- 底部执行反馈默认只显示一条状态；展开后作为大排查面板使用。

## 参考方向

参考大厂 AI 视频工具的工作台式布局，但不照抄：

- 左栏：项目、任务、流程状态
- 中间：当前任务主编辑区
- 右侧或卡片区：上下文、版本、问题、结果

当前原型采用工作台壳子：左侧导航 + 主区 + 右侧覆盖工具抽屉。
