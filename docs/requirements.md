# 需求方案

## 目标

构建一个面向长剧本短剧生产的 Web 系统。用户输入全集剧本后，系统辅助完成：

- 剧本格式清洗与审校
- 资产提取与资产描述
- 资产生图与版本锁定
- 剧本统筹、分镜统筹和视频提示词分层生成
- 分块规划生成与复核
- 视频提示词生成
- 多格式交付导出

最终目标只有两类产物：

- 资产描述/资产生图提示词
- 分镜统筹/视频提示词

其他产物都是中间产物，必须可靠、有效、精准，并且能被后续阶段明确使用。

## 核心约束

- 全剧剧本可达 50000 字或更长。
- 剧本包含多集、多场、多角色、多道具。
- 分镜不能简单逐段翻译剧本，必须依赖全剧、单集、场景、角色、道具、镜头连续状态。
- 系统必须适应不同题材、导演风格、输出格式、规则包和技能包。
- 所有关键产物必须可审阅、可编辑、可追踪、可导出。

## 需求编号

### R-001 剧本源头确认

系统必须支持剧本清洗稿，并让用户直观看到 LLM 后续使用的信息源头。

包括：

- 原始剧本输入
- 格式清洗稿
- 疑似错字/格式混乱/对白异常/角色名混用提示
- 清洗稿确认后进入后续流程

### R-002 长文本处理

系统必须能处理 50000 字级别全集剧本。

包括：

- 自动分集
- 自动分场
- 分块处理
- 上下文摘要
- 中间产物缓存
- 局部重跑
- 失败重试

### R-003 资产真源

系统必须生成角色、场景、道具资产真源。

资产必须支持：

- 统一命名
- 描述
- 连续性规则
- 生图提示词
- 候选图
- 锁定图
- 版本记录

### R-004 资产生图工作台

资产描述后续会用于模型生图，因此需要单独页面管理。

包括：

- 资产列表
- 生图提示词
- 生成结果
- 多版本选择
- 锁定版本
- 锁定图进入分镜提示词上下文

### R-005 分镜阶段中间产物

系统必须生成必要中间产物，供分镜和提示词使用。

中间产物必须遵守最小有效原则：下游明确使用、来源可追踪、可靠性可判断。

包括：

- 全剧统筹：叙事结构、角色弧线、伏笔与连续性风险。
- 章节概要：章节概要、每集标题、每集一句话梗概。
- 单集概要：单集概要。
- 场次概要：人物进入状态、潜台词、空间关系、道具状态。
- 分块规划：视频块规划、剧本原文、起止状态和资产引用。
- 视频提示词：视频块提示词草案。

### R-006 分镜生成

分镜生成必须是视听语言设计，不是剧本文字逐段翻译。

分镜生成输入必须包含：

- 当前剧本文字
- 单集概要
- 可选场次概要
- 必要章节概要
- 资产真源索引
- 当前场次概要、资产索引和相邻视频块状态
- 风格规则

首次生成以场级为主。当前分镜阶段先实现 单集概要/场次概要/分块规划 的场级执行和重跑，不设置独立块级分镜节点。

### R-007 风格与规则包

系统必须支持不同题材、导演风格、规则包、技能包。

包括：

- 题材 profile
- 导演 profile
- 剧本清洗规则包
- 资产规则包
- 分镜规则包
- 生图提示词规则包
- 视频提示词规则包
- 校验规则包

### R-008 输出适配

系统必须支持多种输出格式。

当前默认：

- `assets.json`
- `shots.json`
- `prompts.json`
- `grouped_prompts.csv`
- 项目 ZIP

后续通过交付适配器扩展。

### R-009 可追踪与可溯源

所有需求、设计决策、代码模块和实现状态必须有追踪关系。

包括：

- 需求编号
- 设计决策编号
- 代码模块
- 实现状态
- 后续风险

### R-010 项目管理与保存

系统必须支持多个项目、保存和版本。

P0 阶段先支持：

- 本地项目列表
- 新建项目
- 保存版本摘要
- 当前项目状态恢复

后续支持：

- 后端持久化
- 完整版本 diff
- 回滚
- 权限

### R-011 工作流执行追踪

系统必须记录每次节点运行。

包括：

- 节点运行记录
- 节点产物元信息
- 节点输入/输出引用
- 节点状态
- 阻塞原因
- 执行日志

### R-012 执行反馈与开发排查

系统底部必须有执行反馈区。

- 默认收起为窄状态栏
- 展开后为大排查面板
- 支持搜索和问题过滤
- 后续接 LLM 调用日志、任务队列日志、组件函数日志

### R-023 竖屏短剧 LLM 分层工作流

系统必须支持一套可配置的竖屏短剧默认 LLM 工作流。

默认层级：

- 全剧统筹
- 章节概要
- 单集概要
- 场次概要
- 分块规划
- 视频提示词

要求：

- 全部关键节点默认使用 LLM。
- 全剧统筹允许拆成 剧情地图、角色概要、信息连续性。
- 章节概要负责输出章节概要、每集标题、每集一句话梗概。
- 单集概要负责输出单集概要。
- 场次概要可独立执行，也可合并到 05。
- 分块规划默认按场执行。
- 视频提示词按视频块或区间执行。
- 每层只输出下一层看不到、但必须使用的信息。

详见：[竖屏短剧 LLM 分层工作流需求设计](./vertical-short-drama-workflow.md)。

### R-024 一级页面重构

系统一级页面必须符合人类连续操作习惯。

确认后的一级页面：

- 项目管理
- 剧本校验
- 剧本统筹
- 资产审阅
- 分镜统筹
- 视频生成
- 任务记录
- 流程配置

要求：

- 取消独立“辅助信息”一级页面。
- 剧本统筹承载全剧统筹和章节概要。
- 资产审阅承载资产真源、版本、图片真源。
- 分镜统筹承载 单集概要/场次概要/分块规划。
- 视频生成承载视频提示词、资产包、视频候选、人工确认。
- 不允许让用户在一个一级页面里做到一半，再被迫跳到另一个页面继续同一任务。
