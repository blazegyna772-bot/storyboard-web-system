# 开发流程约定

后续开发按以下顺序执行：

1. 更新或新增需求编号。
2. 更新产品/交互设计。
3. 更新架构或决策记录。
4. 更新追踪矩阵。
5. 再进入代码实现。
6. 实现后更新状态和风险。

## 代码实现要求

- 新增页面必须对应需求编号。
- 新增核心数据结构必须记录在架构文档。
- 新增流程阶段必须进入 Pipeline 设计。
- 新增导出格式必须作为 Output Adapter 记录。
- 新增规则必须作为 Rule Pack 记录。

## 状态标记

- `未完成`
- `原型完成`
- `部分完成`
- `已完成`
- `废弃`
