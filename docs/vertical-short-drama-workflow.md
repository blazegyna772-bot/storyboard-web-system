# 竖屏短剧 LLM 分层工作流需求设计

## 目标

本阶段先确认竖屏短剧的一套默认 LLM 工作流。工程实现、页面设计、规则包加载、任务队列都应服务这套分层逻辑，而不是先从工程节点倒推业务。

适用项目：

- 竖屏短剧，约 60-80 集。
- 单集约 500-1000 字。
- 单集通常 1-3 个物理场次。
- 最终产物仍然是资产描述/生图提示词、分镜规划/视频提示词。

核心原则：

- 全部关键分析节点默认使用 LLM。
- 每一层只输出下一层看不到、但必须使用的信息。
- 中间产物必须可靠、简洁、可审阅、可修改、可重跑。
- 每层都有明确禁区，避免重复分析和无用信息堆积。
- 工作流未来必须可按项目类型替换，例如横屏短剧、电影、广告等可以加载不同分层配置。

## 层级总览

| 编号 | 层级 | 调用单位 | 默认调用 | 核心产物 |
|---|---|---|---|---|
| 01 | 全剧层 | 全剧 | 1 次或多次输出 | 叙事圣经、章节节点图、角色弧线、伏笔、资产变化风险 |
| 02 | 章节层 | 章节 | 按章节调用 | 章节任务卡、每集标题、每集一句话梗概 |
| 03 | 单集层 | 单集 | 每集调用 | 单集任务卡 |
| 04 | 场次简报层 | 单场 | 单独执行或合并到 05 | 导演简报 |
| 05 | 分镜执行层 | 单场 | 每场调用 | 场内分镜脚本 |
| 06 | 视频提示词层 | 镜头或小批镜头 | 按镜头/小批量调用 | 视频模型提示词 |

## 01 全剧层

### 为什么需要

章节层只看本章节剧本，看不到全剧终点、角色完整弧线、后文 callback、关键资产变化风险。01 的作用是建立全剧坐标，告诉下游整部戏在讲什么、角色要去哪里、哪些信息后面会回收。

### 输入

- 全剧完整剧本。
- 项目类型：竖屏短剧。
- 若全剧超出模型稳定处理范围，可分段提炼后汇总。

### 输出

01 不建议一次性让 LLM 输出所有内容。由于输入庞大、输出类型多，准确性不稳，拆成 3 类子调用：

| 子节点 | 名称 | 输入 | 输出 | 作用 |
|---|---|---|---|---|
| 01A | 全剧叙事结构 | 全剧剧本 | 主线、类型基调、全剧情绪曲线、章节节点图、关键转折 | 建立全剧骨架 |
| 01B | 角色弧线与关系 | 全剧剧本 + 01A | 主要角色档案、核心欲望、致命缺陷、弧光终点、关系变化、身份变化 | 专注人物，不被伏笔/资产分散 |
| 01C | 伏笔与连续性风险 | 全剧剧本 + 01A + 可选资产真源 | 伏笔/callback、视觉母题、关键道具、反复空间、服装/道具/场景状态变化风险 | 给章节、单集、分镜提供关注点 |

可选 01D 汇总：

- 输入：01A、01B、01C。
- 输出：面向 02 章节层的全剧叙事圣经总包。
- 规则：只整合，不重新发明内容。

### 字段范围

01A 全剧叙事结构：

- 一句话梗概。
- 类型/基调标签。
- 全剧情绪曲线。
- 章节节点图。
- 关键转折集。

01B 角色弧线与关系：

- 主要角色档案。
- 核心欲望。
- 致命缺陷。
- 弧光终点。
- 关系变化。
- 身份/立场变化。

01C 伏笔与连续性风险：

- 伏笔/callback 清单。
- 核心主题与视觉母题。
- 关键道具。
- 反复出现的重要空间。
- 服装变化风险。
- 道具状态变化风险。
- 场景状态变化风险。
- 身份外观变化风险。

### 禁区

- 不写分镜。
- 不写单集任务。
- 不定最终资产名。
- 不做镜头级建议。
- 不输出每集标题和每集一句话梗概。

每集标题和每集一句话梗概放在 02 章节层，因为 02 看到的是本章节剧本，能更稳定地总结章节内每一集。

## 02 章节层

### 为什么需要

03 单集层只看一集和前后集，无法知道这一组集共同服务什么章节任务。02 的作用是把 01 的全剧坐标落到具体章节，确定一组集共同承担的戏剧功能。

### 输入

- 本章节剧本。
- 01 全剧层提炼信息。
- 01 的章节节点图。
- 必要的前后章节摘要。

### 输出

章节任务卡：

- 章节名称。
- 起止集数。
- 章节功能。
- 情绪主调。
- 必须出现的母题/伏笔。
- 章节结束钩子。
- 本章节每集标题。
- 本章节每集一句话梗概。

### 禁区

- 不写镜头。
- 不做单集详细任务卡。
- 不生成视频提示词。
- 不修改资产真源。

### 人工审阅重点

- 章节功能是否准确。
- 情绪主调是否符合全剧阶段。
- 章节结束钩子是否明确。
- 每集标题是否能准确表达该集核心内容。
- 每集一句话梗概是否能给 03 单集层提供上下文。

## 03 单集层

### 为什么需要

竖屏短剧中，每集是传播单元、钩子单元、情绪单元。分镜层如果只看本集剧本，会平均处理台词，不知道本集服务哪个钩子，也不知道开场和结尾应该如何承接上下文。

### 输入

- 本集剧本。
- 所属 02 章节任务卡。
- 本章节每集标题和每集一句话梗概。
- 前后各两集简要剧情。
- 01 全剧层中必要的角色、伏笔、资产变化风险。

### 输出

单集任务卡：

- 本集一句话任务。
- 情绪标签：开场情绪 -> 结尾情绪。
- 钩子类型：信息反转、情感共鸣、悬念、爽点爆发等。
- 必须用镜头放大的细节：微表情、道具特写、关键台词、关键动作等。
- 节奏指令：极快切换、片刻窒息停顿、压迫、释放等。
- 承上：接住上一集什么情绪或悬念。
- 启下：交给下一集什么情绪或疑问。
- 本集资产/连续性关注点：服装、道具状态、空间变化、人物状态等。

### 禁区

- 不输出完整镜头表。
- 不输出最终视频提示词。
- 不新增资产真源。
- 不凭空制造剧情爽点。

## 04 场次简报层

### 为什么需要

竖屏短剧每集通常 1-3 场，04 可以单独执行，也可以合并到 05 分镜执行层。它存在的意义是解决复杂场内调度和连续性问题，尤其是多人站位、人物进出、道具状态、空间方位关系。

### 输入

- 本场剧本。
- 03 单集任务卡。
- 前后场简述。
- 必要资产真源。

### 输出

导演简报：

- 本场戏剧任务。
- 角色进入状态。
- 潜台词。
- 必须强调的信息。
- 关键物件/表情/动作。
- 空间方位关系。
- 节奏与氛围。
- 承接与悬念。

### 执行策略

- 简单竖屏集：04 可合并到 05。
- 复杂竖屏集：04 单独调用。
- 多人站位、桌边对话、关键道具状态、复杂空间移动时，优先启用 04。

### 禁区

- 不输出完整分镜。
- 不生成视频提示词。
- 不修改资产真源。

## 05 分镜执行层

### 为什么需要

这一层才真正把剧本和导演指令翻译成视听语言。它必须服从 03 单集任务卡，并在需要时参考 02 章节任务卡和 04 场次简报。

### 输入

- 本场完整剧本。
- 03 单集任务卡。
- 可选 02 章节任务卡。
- 可选 04 场次简报。
- 已确认资产真源。
- 必要上一场状态。

### 输出

场内全部分镜脚本：

- 镜号。
- 景别。
- 角度与运动。
- 画面描述：动作、表情、环境。
- 对白/旁白。
- 音效与音乐提示。
- 转场方式。
- 角色资产引用。
- 场景资产引用。
- 道具资产引用。
- 连续性要求。
- 钩子镜头标记。

### 执行策略

- 竖屏按场执行。
- 单集内多场必须保持连续。
- 不满意时支持单场重跑。
- 重跑时锁定资产真源、03 单集任务卡、必要连续性状态。

### 禁区

- 不输出最终视频模型提示词。
- 不修改资产真源。
- 不重写 03 单集任务。

## 06 视频提示词层

### 为什么需要

分镜脚本不是视频模型可直接执行的提示词。06 把已确认分镜、资产图和模型规则转成视频模型语言。

### 输入

- 最终分镜。
- 资产真源图。
- 视频模型规则。
- 画幅/模型参数。

### 输出

- 正向提示词。
- 参考图路径。
- 时长。
- 画幅。
- 镜头运动。
- 动作连续性。
- 负面提示词。
- 模型参数。

### 禁区

- 不改剧情。
- 不重排分镜。
- 不更改资产。

## 默认数据流

```text
01 全剧剧本
  -> 01A 全剧叙事结构
  -> 01B 角色弧线与关系
  -> 01C 伏笔与连续性风险
  -> 01D 全剧叙事圣经汇总（可选）

02 本章节剧本 + 01 提炼信息
  -> 章节任务卡 + 每集标题 + 每集一句话梗概

03 本集剧本 + 02 章节任务卡 + 前后集摘要
  -> 单集任务卡

04 本场剧本 + 03 单集任务卡
  -> 场次导演简报（可单独执行或合并到 05）

05 本场剧本 + 03 单集任务卡 + 可选 04 场次简报
  -> 场内分镜脚本

06 分镜 + 资产真源图 + 视频模型规则
  -> 视频提示词
```

## 对工程设计的要求

后续工程实现必须支持：

- 项目类型驱动工作流配置。
- 节点可热插拔。
- 每个节点的输入、输出、提示词、模型参数、重跑粒度可配置。
- 每个节点产物可审阅、可编辑、可保存、可追踪。
- 01 层允许多子节点输出，并允许汇总节点。
- 04 层允许按配置决定独立执行或合并到 05。
- 05 层默认按场执行。
- 02 层负责每集标题和每集一句话梗概，不放在 01 层。
