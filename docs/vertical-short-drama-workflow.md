# 竖屏短剧 LLM 分层工作流需求设计

## 目标

本阶段确认竖屏短剧的一套默认 LLM 工作流。工程实现、页面设计、规则包加载、任务队列都应服务这套分层逻辑，而不是先从工程节点倒推业务。

适用项目：

- 竖屏短剧，约 60-80 集。
- 单集约 500-1000 字。
- 单集通常 1-3 个物理场次。
- 最终产物仍然是资产描述/生图提示词、分镜设计、视频提示词。

核心原则：

- 全部关键分析节点默认使用 LLM。
- 每一层只输出下一层看不到、但必须使用的信息。
- 中间产物必须可靠、简洁、可审阅、可修改、可重跑。
- 每层都有明确禁区，避免重复分析和无用信息堆积。
- 工作流未来必须可按项目类型替换，例如横屏短剧、电影、广告等可以加载不同分层配置。
- 节点名不使用数字。执行顺序由工作流配置决定，数字只用于集数、场次、镜号、版本等实例编号。

## 节点总览

| 执行顺序 | 节点 | 机器 ID | 调用单位 | 默认调用 | 核心产物 |
|---|---|---|---|---|---|
| 1 | 剧情地图 | `story_map` | 全剧 | 单次或分段 | 剧情大纲、章节节点图、关键转折、剧情地图切换 |
| 2 | 角色概要 | `character_summary` | 全剧 | 单次或分段 | 角色功能、身份/关系/认知变化、视觉身份风险 |
| 3 | 信息连续性 | `continuity` | 全剧 | 单次或分段 | 伏笔、callback、视觉母题、跨集状态风险 |
| 4 | 全集概要 | `series_summary` | 全剧 | 程序机械合并 | 面向下游的全剧总入口，不做剧情理解 |
| 5 | 章节概要 | `chapter_summary` | 章节 | 按章节调用 | 章节功能、情绪主调、章节钩子、每集标题和一句话梗概 |
| 6 | 单集概要 | `episode_summary` | 单集 | 每集调用 | 单集任务、情绪变化、钩子、必须放大的细节 |
| 7 | 场次概要 | `scene_summary` | 单场 | 可独立执行 | 场内调度、角色进入状态、潜台词、空间关系 |
| 8 | 分镜设计 | `storyboard_design` | 单场 | 每场调用 | 场内镜头表、空间时序、资产引用、连续性要求 |
| 9 | 视频提示词 | `video_prompt` | 镜头或小批镜头 | 按镜头/小批量调用 | 视频模型提示词、参考图路径、模型参数 |

页面归属：

- 剧本统筹：剧情地图、角色概要、信息连续性、全集概要、章节概要。
- 分镜统筹：单集概要、场次概要、分镜设计。
- 视频生成：视频提示词。

## 全剧统筹节点

### 为什么需要

章节概要只看当前章节剧本，看不到全剧终点、角色完整弧线、后文 callback、关键资产变化风险。全剧统筹节点的作用是建立全剧坐标，告诉下游整部戏在讲什么、角色要去哪里、哪些信息后面会回收。

### 剧情地图

输入：

- 全剧完整剧本。
- 项目类型：竖屏短剧。

输出：

- 一句话梗概。
- 类型/基调标签。
- 全剧剧情大纲，覆盖主要章节推进。
- 关键转折地图。
- 章节节点图。
- 章节主要空间与剧情地图切换。

禁区：

- 不写分镜。
- 不写单集任务。
- 不定最终资产名。
- 不输出每集标题和每集一句话梗概。

### 角色概要

输入：

- 全剧完整剧本。
- 剧情地图。

输出：

- 主要角色功能。
- 核心欲望。
- 身份/处境/外观状态变化节点。
- 关系变化节点。
- 认知状态变化节点。
- 极短表演底色。
- 后续制作风险。

说明：这里的“角色”包含人、动物、妖怪等所有承担戏剧功能的主体。

禁区：

- 不做角色外观资产真源。
- 不写生图提示词。
- 心理分析必须压缩，优先写会影响后续制作判断的信息。

### 信息连续性

输入：

- 全剧完整剧本。
- 剧情地图。

输出：

- 伏笔/callback 清单。
- 核心主题与视觉母题。
- 关键道具风险。
- 反复出现的重要空间。
- 角色服装/身份变化风险。
- 道具状态变化风险。
- 场景状态变化风险。
- 需要人工复核的章节或集。

禁区：

- 不做资产最终命名。
- 不写分镜。
- 不写视频提示词。

### 全集概要

全集概要由后端程序机械合并剧情地图、角色概要、信息连续性生成。

规则：

- 不理解剧情。
- 不判断冲突。
- 不重写内容。
- 不排序筛选。
- 只搬运固定字段。

用途：作为章节概要的默认全剧输入入口。原始剧情地图、角色概要、信息连续性仍可用于人工追溯。

## 章节概要

### 为什么需要

单集概要只看一集和前后集，无法知道这一组集共同服务什么章节任务。章节概要的作用是把全集概要落到具体章节，确定一组集共同承担的戏剧功能。

### 输入

- 当前章节剧本。
- 全集概要。
- 当前章节相关的信息连续性风险。

后端必须按剧情地图里的 `chapter_map.episode_range` 切出当前章节剧本，不允许把全集剧本直接塞给章节概要。

### 输出

- 章节名称。
- 起止集数。
- 章节功能。
- 情绪主调。
- 必须出现的母题/伏笔。
- 章节结束钩子。
- 本章节每集标题。
- 本章节每集一句话梗概。
- 章节复核风险。

### 禁区

- 不写镜头。
- 不做单集详细任务。
- 不生成视频提示词。
- 不修改资产真源。

## 单集概要

### 为什么需要

竖屏短剧中，每集是传播单元、钩子单元、情绪单元。分镜设计如果只看本集剧本，会平均处理台词，不知道本集服务哪个钩子，也不知道开场和结尾应该如何承接上下文。

### 输入

- 当前集剧本。
- 当前章节概要。
- 前后集概要。
- 必要的信息连续性风险。

### 输出

- 本集一句话任务。
- 情绪标签：开场情绪 -> 结尾情绪。
- 钩子类型：信息反转、情感共鸣、悬念、爽点爆发等。
- 必须用镜头放大的细节：微表情、道具特写、关键台词、关键动作等。
- 节奏指令：极快切换、片刻窒息停顿、压迫、释放等。
- 承上。
- 启下。
- 本集资产/连续性关注点。

### 禁区

- 不输出完整镜头表。
- 不输出最终视频提示词。
- 不新增资产真源。
- 不凭空制造剧情爽点。

## 场次概要

### 为什么需要

竖屏短剧每集通常 1-3 场。场次概要可以单独执行，也可以在简单场景里跳过。它存在的意义是解决复杂场内调度和连续性问题，尤其是多人站位、人物进出、道具状态、空间方位关系。

### 输入

- 当前场剧本。
- 当前单集概要。
- 前后场简述。
- 必要资产真源。

### 输出

- 本场戏剧任务。
- 角色进入状态。
- 潜台词。
- 必须强调的信息。
- 关键物件/表情/动作。
- 空间方位关系。
- 节奏与氛围。
- 承接与悬念。

### 执行策略

- 简单竖屏集：可跳过。
- 复杂竖屏集：单独调用。
- 多人站位、桌边对话、关键道具状态、复杂空间移动时，优先启用。

### 禁区

- 不输出完整分镜。
- 不生成视频提示词。
- 不修改资产真源。

## 分镜设计

### 为什么需要

这一层才真正把剧本和导演指令翻译成视听语言。它必须服从当前单集概要，并在需要时参考当前章节概要和当前场次概要。

### 输入

- 当前场完整剧本。
- 当前单集概要。
- 可选当前章节概要。
- 可选当前场次概要。
- 已确认资产真源。
- 必要上一场状态。

### 输出

- 镜号。
- 景别。
- 角度与运动。
- 画面描述：动作、表情、环境。
- 对白/旁白。
- 音效与音乐提示。
- 转场方式。
- 角色资产引用。
- 场景资产引用。
- 道具资产引用。
- 连续性要求。
- 钩子镜头标记。
- 必要场内空间时序。

### 执行策略

- 竖屏按场执行。
- 单集内多场必须保持连续。
- 不满意时支持单场重跑。
- 重跑时锁定资产真源、当前单集概要、必要连续性状态。

### 禁区

- 不输出最终视频模型提示词。
- 不修改资产真源。
- 不重写当前单集概要。

## 视频提示词

### 为什么需要

分镜设计不是视频模型可直接执行的提示词。视频提示词节点把已确认分镜和资产真源转成视频模型语言；通用模型规则写在提示词模板内。

### 输入

- 当前分镜设计。
- 资产真源。

### 输出

- 视频组提示词。
- 参考图路径。
- 时长。
- 草稿状态。
- 视频路径。

### 禁区

- 不改剧情。
- 不重排分镜。
- 不更改资产。

## 默认数据流

```text
全剧剧本
  -> 剧情地图
  -> 角色概要
  -> 信息连续性
  -> 全集概要（程序机械合并）

当前章节剧本 + 全集概要
  -> 章节概要 + 每集标题 + 每集一句话梗概

当前集剧本 + 当前章节概要 + 前后集摘要
  -> 单集概要

当前场剧本 + 当前单集概要
  -> 场次概要（可单独执行或跳过）

当前场剧本 + 当前单集概要 + 可选当前场次概要
  -> 分镜设计

当前分镜设计 + 资产真源
  -> 视频提示词
```

## 对工程设计的要求

后续工程实现必须支持：

- 项目类型驱动工作流配置。
- 节点可热插拔。
- 每个节点的输入、输出、提示词、模型参数、重跑粒度可配置。
- 每个节点产物可审阅、可编辑、可保存、可追踪。
- 全剧统筹允许多子节点输出，并允许程序机械汇总。
- 场次概要允许按配置决定独立执行或跳过。
- 分镜设计默认按场执行。
- 章节概要负责每集标题和每集一句话梗概，不放在剧情地图里。
