# UI 布局方案

## 目标

当前阶段的 UI 目标不是视觉包装，而是让系统像一个可工作的生产工具：

- 信息层级清楚
- 当前页面任务明确
- 主操作容易找到
- 长内容可审阅
- 状态、锁定、重跑、版本后续可扩展

## 参考图

UI 改版参考图统一保存在 `docs/ui-references/`，索引见 `docs/ui-references/README.md`。这些图只用于参考信息层级、间距、边框、操作流和视觉秩序；具体字段、功能和状态仍以本项目真实业务为准。

## 总体布局

采用桌面优先的工作台布局：

```text
顶部窄栏：当前项目、保存/生成/导出、状态、全局工具入口
左侧窄导航：一级模块，可收起
中间主区：当前任务的主审阅和编辑区域
右侧工具抽屉：默认收起，打开后覆盖在页面上层，不挤压主区
```

## 左侧导航规则

- 左侧只做模块导航，不重复顶部操作。
- 左侧默认窄宽，可收起为图标栏。
- 项目列表入口放左侧，但项目管理内容不塞进左侧。

## 顶部栏规则

- 顶部只放项目名、草稿状态、保存、生成、导出、工具入口。
- 顶部不做页面标签导航。
- 顶部高度必须保持很窄，不占主工作区。

## 右侧工具区规则

- 默认收起。
- 打开后作为覆盖抽屉显示，不改变主工作区宽度。
- 放全局工具：项目管理、剧集跳转、任务状态、导出、诊断等。

## 右主区规则

- 每个页面必须有明确页面标题和说明。
- 每个页面顶部优先显示状态/统计。
- 主内容分区应直接服务任务，不堆无效说明。
- 长列表使用自然页面滚动，避免多个内部滚动区域互相嵌套。

## 页面规划

### 剧本校验

主区：

- 剧本输入
- 参数
- 导入/生成/导出
- 清洗稿
- 审校问题
- 写回剧本

### 剧本统筹

主区：

- 全剧统筹运行状态和产物审阅。
- 剧情地图。
- 角色概要。
- 信息连续性。
- 章节概要运行状态和产物审阅。
- 章节概要。
- 每集标题。
- 每集一句话梗概。

这个页面是资产审阅前置页。它不写分镜，不做视频提示词。

### 资产审阅

右侧：

- 生图队列
- 资产描述编辑
- 生图提示词编辑
- 首次来源和连续性
- 候选图/锁定图
- 失败原因和待处理状态

不要只做资产展示列表。这个页面的人类任务是确认描述、发起生图、比较候选、锁定版本。

### 分镜统筹

右侧：

- 单集概要。
- 场次概要。
- 分块规划。
- 本场引用资产。
- 场级重跑和锁定。
- 分镜审阅。

正式生产状态下，该页面应在资产真源确认后执行，单集概要、场次概要、分块规划必须使用资产真源 ID 和版本。当前工程阶段先不强制衔接资产审阅，页面以分镜工作流验证为主。

### 视频生成

右侧：

- 视频提示词。
- 视频块资产包。
- 资产真源图引用。
- 视频候选。
- 人工确认。
- 单块重跑。

这个页面服务最终人工检查。它更偏视觉化，重点是提示词、资产包、视频结果，而不是全剧/章节/单集文字逻辑。

### 流程配置

右侧：

- Profile
- 提示词模板
- Providerpack / Rulepack
- 工作流节点
- 节点输入、节点产物、重跑粒度

流程配置不是日常生产页。它用于项目前配置、换体裁/规则包、排查某阶段异常。

边界说明：前端只保留“提示词模板”作为 Prompt/Rulepack 的正式管理入口；后端 Rulepack 扫描是模板库的数据来源，不再单独做一个与模板库重复的“后端规则”页面。LLM 调用日志归入“文本 API”配置页，作为接口诊断信息。

## 后续 UI 需求

- 资产页增加筛选：角色/场景/道具/已锁定/待生成。
- 分镜统筹页增加时间线模式。
- 视频生成页增加候选视频对比和视频块确认状态。
- Pipeline 页增加阶段运行状态和产物查看。
- 所有页面增加锁定状态和版本历史入口。

### 项目管理页后处理事项

这些事项不属于当前 UI 布局阶段，进入 UI 大阶段之后的程序/数据整理阶段再处理：

维护规则：UI 阶段中凡是判断为“后处理/程序数据阶段再做”的事项，必须随手追加到本节或对应页面的后处理事项中，不等额外提醒。

- 项目封面存储方式：已改为上传到项目目录 `assets/project/`，`project.json` 保存封面 URL；旧项目里已有的 data URL 仍可兼容显示。
- 项目元数据字段：已补 `createdAt / description / owner / status / coverImage` 默认字段，兼容旧项目读取。
- 创建时间排序：已接入 `createdAt`，旧项目缺失时回退 `updatedAt`。

后续保留事项：

- 默认封面规则：当前默认取角色真源第一张图；后续应明确优先主角、已确认/已锁定资产，或支持人工指定资产为项目封面。
- 项目元数据编辑：项目描述、负责人、状态已具备字段，但还缺专门编辑/保存入口。
- 项目参数权限：开工后哪些参数锁定、哪些允许改、是否需要二次确认，需要作为业务规则处理。
- 项目分页/筛选：当前是前端分页和搜索；项目数量大后再改为后端分页、排序和筛选。
- 项目状态：已具备 `status` 字段；后续再做筹备、制作中、返修、完成、归档等状态编辑、筛选和持久化规则。
- 切换项目策略：当前已提示“单击查看、双击设为当前项目”；是否需要误切换确认、加载提示或大项目加载策略，后续再定。

### 剧本校检页后处理事项

当前已落地：

- 两阶段流程：`导入与分集确认 / 正式校检` 已在一级页面内完成，导入全集不再走旧弹窗。
- 分集确认：支持全集导入、分集预览、当前集文本编辑、稳定异常提示、确认分集并保存到当前项目。
- 正式校检：支持按集编辑、行号显示、问题定位、忽略/已处理/重开、本地状态持久化、问题导出。
- 规则配置：分集规则和正则校验规则已拆分为右侧配置面板，并只保留稳定程序规则。
- 分集稳定检查：已支持缺集明确提示、重复集、集号顺序异常、字数过短/过长/明显偏离平均值。
- 正则稳定检查：已支持场次标题格式、场次号连续性、场次归属集号、空场次、连续重复行、括号/引号未闭合、明显占位符、非法字符、格式残留符号。
- 保存入口已拆分：`保存人工编辑稿` 写回当前人工编辑内容，`应用规则清洗稿` 写回规则清洗结果，避免误把规则清洗稿当普通保存。

后续保留事项：

- 语义类校检：本版本不做。人物关系、剧情逻辑、前后文一致性、角色名相似等正则无法稳定保障的检查，不进入本页程序规则。
- 非连续重复内容：只检查相邻完全重复行；跨段重复、重复台词是否合理不做程序判断。
- 审校推荐处理：如果后续需要“推荐修改 -> 人工确认 -> 写入剧本”的闭环，需要单独设计推荐数据结构和写入策略。
- 原始稿/编辑稿双区：如要做只读原始稿与可编辑校检稿并排联动，需要重构文本区域组件和滚动同步策略。
- 保存语义深化：当前已拆分人工编辑稿和规则清洗稿；如后续增加审校推荐稿，需要新增推荐修订版本与确认规则。
- 分集边界手动调整：当前支持全集导入、分集预览、当前集文本编辑、单集追加/替换和确认分集；尚未提供专门的“向上一集合并 / 向下一集拆分 / 移动边界”控件，后续如需要精细调边界再补。
- 汇总指标范围：正式校检顶部当前混合了全剧指标和当前集问题指标；后续应明确分区或标注“全剧 / 当前集”，避免制作时误读。
- 长文本编辑器压测：当前真实行号编辑器使用逐行 textarea，解决了模拟行号错位问题；后续需要用完整长剧本压测滚动、定位、输入延迟，如不足再考虑虚拟列表或专门文本编辑组件。
- 问题状态继承：当前问题状态按项目和集号本地保存；重新校检后若文本大幅变化，后续可设计状态失效或继承策略。
- 集数右键注释：当前保留原生 `title`；如需自定义注释菜单，后续统一设计，不在本页程序规则阶段展开。

### 剧本统筹 / 分镜统筹页后处理事项

当前已落地：

- 剧本统筹页已移除“运行分镜全流程”入口，只保留本页统筹执行、节点执行和刷新。
- 剧本统筹页已使用独立页面语义：全剧信息流、汇总产物、当前统筹产物、全剧基础信息、全集与章节。
- 剧本统筹页已改为两阶段结构：`全集概要 / 章节概要`。全集概要阶段使用横排标签页：`全集概要 / 剧情地图 / 角色概要 / 信息连续性`；章节概要阶段使用章节横排标签和对应章节产物区。
- 剧本统筹页顶部概览区已移除，`已完成 / 待关注` 统计收进标题栏。
- 分镜统筹页已改为三阶段节点结构：`单集概要 / 场次概要 / 分块规划`，默认聚焦 `分块规划`，单集和场次概要作为前置信息产物查看。
- 分镜统筹页切换集时会校正当前场次，避免选中的场次不属于当前集。
- 分镜/视频关键产物已改为按集场隔离存储：`episode_summary/EPxx.json`、`scene_summary/EPxx_SCxx.json`、`storyboard_design/EPxx_SCxx.json`、`video_prompt/EPxx_SCxx.json`；旧节点单文件只作为兼容读取。
- 分镜统筹页和视频生成页已按当前集/场主动读取 scoped 产物，避免不同场次的分块规划、场次概要和视频提示词互相覆盖。
- 分镜统筹页已增加 `本集全部场次` 执行入口，按当前集所有场次串行执行当前模式下的分镜统筹节点，并写入各自集场产物。

后续保留事项：

- Prompt 收紧记录：本轮测试中 `character_summary` 出现小范围时间归属漂移，例如“团团 EP05-EP10 获赠夜明珠等礼物”与连续性产物中“夜明珠第12集赠送”不完全一致。当前输出质量已满足生产，不在本轮修改；后续收紧 `story_workflow_character_summary` 时应强调角色状态不要把后文道具或礼物提前归入前一阶段，时间范围以剧本明确信息为准。
- 分镜统筹后续可增加更细的批量任务队列、失败后续跑、按场次汇总查看；当前已具备本集全部场次串行执行能力。
- 剧本统筹状态指标：当前已做基础文案参数化；后续可进一步收紧为“全剧层、汇总层、章节层”的生产完成度和可继承产物状态。
- 分镜统筹状态指标：后续应单独围绕当前集/场/视频块前置产物设计指标，不应完全沿用剧本统筹的全剧节点指标。
- 分块规划产物后续需要做专用审阅界面，包括块列表、时长、资产锚定、剧本真源片段和人类检查状态；当前阶段先沿用通用产物审阅。

### 资产审阅页后处理事项

当前已落地：

- `按章节提取资产` 已改为按章节串行执行三类新版 Prompt：角色、场景、道具。每章按角色 -> 场景 -> 道具顺序运行，并把 `records` 与 `true_sources` 覆盖写入当前项目。
- 已收紧按章资产 Prompt：`测试剧集0620` 暴露的版本判断问题已落到新版 Prompt。全局资产线索现在要求同时参考 `character_summary`、`continuity`、`series_summary`、`chapter_summaries`。
- 已补程序保护：按章资产合并端对同名道具/场景、同基础角色名做弱归并兜底，避免同章内“夜明珠完整版/破碎版”这类明显同物状态被拆成两个资产。

后续保留事项：

- 本版本暂不处理：单独的 `提取角色/场景/道具记录` 仍接旧版全集单类 Prompt，后续是否改名、折叠或隐藏再定。
- 本版本暂不处理：`测试剧集0620` 资产复测中暴露的语义判断细节，例如 `灵儿` 年龄感、灵儿入府/周府后版本、团团王府常服/华丽郡主装/手链版本、`宸王府厢房` 与 `团团房间` 归并、夜明珠多版本等，后续如继续提升资产提取质量再复测。
