# UI 布局方案

## 目标

当前阶段的 UI 目标不是视觉包装，而是让系统像一个可工作的生产工具：

- 信息层级清楚
- 当前页面任务明确
- 主操作容易找到
- 长内容可审阅
- 状态、锁定、重跑、版本后续可扩展

## 总体布局

采用桌面优先的工作台布局：

```text
顶部窄栏：当前项目、保存/生成/导出、状态、全局工具入口
左侧窄导航：一级模块，可收起
中间主区：当前任务的主审阅和编辑区域
右侧工具抽屉：默认收起，打开后覆盖在页面上层，不挤压主区
```

## 左侧导航规则

- 左侧只做模块导航，不重复顶部操作。
- 左侧默认窄宽，可收起为图标栏。
- 项目列表入口放左侧，但项目管理内容不塞进左侧。

## 顶部栏规则

- 顶部只放项目名、草稿状态、保存、生成、导出、工具入口。
- 顶部不做页面标签导航。
- 顶部高度必须保持很窄，不占主工作区。

## 右侧工具区规则

- 默认收起。
- 打开后作为覆盖抽屉显示，不改变主工作区宽度。
- 放全局工具：项目管理、剧集跳转、任务状态、导出、诊断等。

## 右主区规则

- 每个页面必须有明确页面标题和说明。
- 每个页面顶部优先显示状态/统计。
- 主内容分区应直接服务任务，不堆无效说明。
- 长列表使用自然页面滚动，避免多个内部滚动区域互相嵌套。

## 页面规划

### 剧本校验

主区：

- 剧本输入
- 参数
- 导入/生成/导出
- 清洗稿
- 审校疑点
- 写回剧本

### 资产生图

右侧：

- 生图队列
- 资产描述编辑
- 生图提示词编辑
- 首次来源和连续性
- 候选图/锁定图
- 失败原因和待处理状态

不要只做资产展示列表。这个页面的人类任务是确认描述、发起生图、比较候选、锁定版本。

### 上下文包 / 跨段辅助

右侧：

- 03 集级辅助：只显示 06/07 当前边界看不到但必须遵守的信息
- 04 场级连续性：显示场内人物进出、道具流转、空间边界
- 角色/场景/道具锁定项
- 镜头级连续性监看

每条辅助信息必须标明用途、来源、可靠性。不能把重复剧本文字堆在这里。

### 流程配置

右侧：

- Profile
- Rule Pack
- Output Adapter
- Pipeline 阶段
- 粒度、产物角色、锁定策略、重跑粒度

流程配置不是日常生产页。它用于项目前配置、换体裁/规则包、排查某阶段异常。

### 分镜工作台

右侧：

- 本集必须遵守的 03 信息
- 场级规划和连续性
- 重跑控制：场级、块级、只重写提示词
- 镜头审阅
- 视频提示词预览
- JSON 真源编辑

主区不再重复剧集选择和泛统计。选集在左侧完成，主区只处理当前集的分镜审阅与局部重跑。

## 后续 UI 需求

- 资产页增加筛选：角色/场景/道具/已锁定/待生成。
- 分镜页增加时间线模式。
- Pipeline 页增加阶段运行状态和产物查看。
- 所有页面增加锁定状态和版本历史入口。
