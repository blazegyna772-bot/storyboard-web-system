# UI 布局方案

## 目标

当前阶段的 UI 目标不是视觉包装，而是让系统像一个可工作的生产工具：

- 信息层级清楚
- 当前页面任务明确
- 主操作容易找到
- 长内容可审阅
- 状态、锁定、重跑、版本后续可扩展

## 总体布局

采用桌面优先的工作台布局：

```text
顶部窄栏：当前项目、保存/生成/导出、状态、全局工具入口
左侧窄导航：一级模块，可收起
中间主区：当前任务的主审阅和编辑区域
右侧工具抽屉：默认收起，打开后覆盖在页面上层，不挤压主区
```

## 左侧导航规则

- 左侧只做模块导航，不重复顶部操作。
- 左侧默认窄宽，可收起为图标栏。
- 项目列表入口放左侧，但项目管理内容不塞进左侧。

## 顶部栏规则

- 顶部只放项目名、草稿状态、保存、生成、导出、工具入口。
- 顶部不做页面标签导航。
- 顶部高度必须保持很窄，不占主工作区。

## 右侧工具区规则

- 默认收起。
- 打开后作为覆盖抽屉显示，不改变主工作区宽度。
- 放全局工具：项目管理、剧集跳转、任务状态、导出、诊断等。

## 右主区规则

- 每个页面必须有明确页面标题和说明。
- 每个页面顶部优先显示状态/统计。
- 主内容分区应直接服务任务，不堆无效说明。
- 长列表使用自然页面滚动，避免多个内部滚动区域互相嵌套。

## 页面规划

### 剧本校验

主区：

- 剧本输入
- 参数
- 导入/生成/导出
- 清洗稿
- 审校疑点
- 写回剧本

### 剧本统筹

主区：

- 全剧统筹运行状态和产物审阅。
- 剧情地图。
- 角色概要。
- 信息连续性。
- 章节概要运行状态和产物审阅。
- 章节概要。
- 每集标题。
- 每集一句话梗概。

这个页面是资产审阅前置页。它不写分镜，不做视频提示词。

### 资产审阅

右侧：

- 生图队列
- 资产描述编辑
- 生图提示词编辑
- 首次来源和连续性
- 候选图/锁定图
- 失败原因和待处理状态

不要只做资产展示列表。这个页面的人类任务是确认描述、发起生图、比较候选、锁定版本。

### 分镜统筹

右侧：

- 单集概要。
- 场次概要。
- 分镜设计。
- 本场引用资产。
- 场级重跑和锁定。
- 分镜审阅。

正式生产状态下，该页面应在资产真源确认后执行，单集概要、场次概要、分镜设计必须使用资产真源 ID 和版本。当前工程阶段先不强制衔接资产审阅，页面以分镜工作流验证为主。

### 视频生成

右侧：

- 视频提示词。
- 镜头资产包。
- 资产真源图引用。
- 视频候选。
- 人工确认。
- 单镜头重跑。

这个页面服务最终人工检查。它更偏视觉化，重点是提示词、资产包、视频结果，而不是全剧/章节/单集文字逻辑。

### 流程配置

右侧：

- Profile
- Rule Pack
- Output Adapter
- Pipeline 阶段
- 粒度、产物角色、锁定策略、重跑粒度

流程配置不是日常生产页。它用于项目前配置、换体裁/规则包、排查某阶段异常。

## 后续 UI 需求

- 资产页增加筛选：角色/场景/道具/已锁定/待生成。
- 分镜统筹页增加时间线模式。
- 视频生成页增加候选视频对比和镜头级确认状态。
- Pipeline 页增加阶段运行状态和产物查看。
- 所有页面增加锁定状态和版本历史入口。
