# 决策记录

## D-001 当前阶段定位

当前阶段定位为：需求方案 + 设计方案 + 可扩展前端原型。

不是生产系统。

## D-002 基础工作台壳子

采用：顶部窄栏 + 左侧可收起导航 + 中间主工作区 + 右侧覆盖式工具抽屉 + 底部执行反馈。

左侧只做一级模块导航，不承载全文剧本、项目总览或重复流程信息。

顶部放项目、保存、生成、导出和工具入口。

右侧工具抽屉默认收起，打开后覆盖页面，不挤压主工作区。

## D-003 上下文包是正式数据层

“上下文包”在产品上改名为“辅助信息审阅”。它不是信息仓库，不能无限扩大。

它必须服务最终产物，并且每条信息都要说明来源、用途和可靠性。

03 只负责 06/07 当前执行边界看不到的大尺度辅助信息。

04 负责场级连续性、人物进出、道具流转、空间关系。

角色/场景/道具状态只有在会被资产、分镜或提示词引用时才进入中间产物。

## D-004 Pipeline 必须现在设计

由于存在 50000 字长文本、多题材、多规则包、多输出格式，系统必须现在设计 Pipeline 骨架。

后期再补会导致重写。

## D-005 当前规则生成只是占位执行器

当前 `scriptQuality`、`storyboard`、`contextPack` 都是规则版占位。

未来应替换或并行接入 LLM executor。

## D-006 Pipeline 骨架先于模型接入

先实现 PipelineStage、RulePack、GenreProfile、DirectorProfile、OutputAdapter 的前端配置与默认骨架，再接入 LLM。

原因：长剧本、多题材、多输出格式会持续扩展，模型接入不能写死在页面组件中。

## D-007 最终产物导向

最终产物只有资产描述/生图提示词、分镜规划/视频提示词。

所有中间产物必须有明确用途和后续引用。无效、重复、无法被执行阶段使用的信息不进入管线。

## D-008 场级首次规划，块级局部重跑

首次分镜规划以一场戏为调用边界，由 06 负责场内空间时序和连续性。

人工对局部结果不满意时，允许块级重跑，但必须锁定集级/场级状态、资产、相邻镜头。

只有改动影响空间、人物进出、道具流转时才场级重跑；影响信息揭露和人物关系时才集级重跑。

## D-009 人物空间站位归属

人物站位不交给 03 发明。

如果剧本中有明确进出场、移动、桌边关系、人物相对位置，06 场级分镜规划应在整场范围内推导 `scene_spatial_timeline`。

07/08 只引用锁定的场级空间时序，不逐镜随意发明站位。

## D-010 中间产物最小有效原则

中间产物必须满足三条：

- 下游明确使用
- 来源可追踪
- 不重复当前执行单元已经能看到的信息

不满足这些条件的信息不进入 Artifact Store。

## D-011 执行反馈形态

底部执行反馈默认是一条窄状态栏，服务普通使用。

展开后是开发/排查大面板，服务 Pipeline、LLM、组件函数和人工修改日志追踪。

## D-012 本地项目先行

P0 阶段先实现 localStorage 本地项目、多项目和版本摘要。

后端、权限、完整 diff、回滚进入后续生产系统阶段。
