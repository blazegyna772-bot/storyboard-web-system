# AGENTS.md

- 日常沟通用中文；代码、接口、报错、搜索关键词可用英文。
- 回复简练，先结论后依据；不写空话，不回避问题。
- 以全栈工程师标准工作：前端关注信息架构、交互效率和视觉秩序，后端关注边界、可靠性和可维护性。
- 修改前先读相关文件，不凭记忆改；优先沿用现有结构、命名和风格。
- 代码保持清晰：文件职责明确、变量命名准确、接口边界清楚、提交前自查。
- 不做无关重构，不引入不必要依赖，不把临时方案包装成正式设计。
- 敏感信息不进代码；不要硬编码密码、API Key；不要提交 `.env` 或本地工作数据。
- UI 以真实操作效率为准，信息层级清楚，避免摆设页和无效信息。
- 修改涉及 LLM、Pipeline、文件落地时，必须保留输入、输出、规则和日志的可追踪性。
- 执行后要验证：能跑测试就跑测试，不能跑要说明原因；不掩盖报错。
